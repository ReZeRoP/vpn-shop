# فروشگاه کانفیگ V2Ray

> English documentation: [README.md](./README.md)

سایت فروش آنلاین کانفیگ با اتوماسیون کامل روی پنل **3x-ui (ثنایی)**، پرداخت کارت به کارت با تحویل آنی، صف بررسی رسید برای مدیر، و گفتگوی عمومی زنده.

## نصب سریع (یک دستور)

```bash
git clone https://github.com/ReZeRoP/vpn-shop.git
cd vpn-shop
bash install.sh
```

اسکریپت نصب: Node و PM2 را نصب می‌کند، چند سوال می‌پرسد (رمز ادمین، آدرس پنل، ...)، دیتابیس را می‌سازد، سایت را build و با PM2 اجرا می‌کند و در صورت تمایل nginx را هم تنظیم می‌کند.

## امکانات

- 🛒 **فروشگاه**: پلن‌های قابل‌مدیریت از پنل ادمین (مدت، حجم، تعداد کاربر، قیمت)
- ⚡ **تحویل آنی**: بعد از ثبت رسید کارت به کارت، کلاینت به‌صورت خودکار روی پنل 3x-ui ساخته و لینک اشتراک + QR تحویل داده می‌شود
- 🧾 **صف بررسی رسید**: رسیدها برای مدیر ارسال می‌شوند؛ رسید جعلی → لغو سرویس با یک کلیک (غیرفعال‌سازی کلاینت روی پنل — برگشت‌پذیر)
- 🛡️ **ضد تقلب**: مبلغ یکتا برای هر سفارش (کد چندتومانی)، تشخیص رسید تکراری (هش تصویر)، سقف سفارش باز، پرچم‌گذاری خودکار کاربران با سابقه رسید جعلی — سفارش‌های مشکوک بدون تحویل آنی به صف بررسی می‌روند
- 💬 **گفتگوی عمومی**: چت زنده با Socket.IO (تاریخچه، ضدـاسپم، حذف پیام و بی‌صدا کردن توسط مدیر)
- 📊 **پنل کاربری**: مصرف زنده از پنل، حجم باقیمانده، انقضا به تاریخ شمسی
- 🎨 طراحی مدرن و مینیمال، تمام‌فارسی و راست‌چین، فونت وزیرمتن (self-hosted)

## پشته فنی

Next.js 16 (App Router) + custom server • Socket.IO • SQLite (better-sqlite3, WAL) + Drizzle ORM • Tailwind v4 • sharp

## راه‌اندازی روی سرور (Ubuntu)

### ۱. پیش‌نیازها

```bash
# Node.js 20+
curl -fsSL https://deb.nodesource.com/setup_22.x | sudo -E bash -
sudo apt install -y nodejs nginx
sudo npm i -g pm2
```

### ۲. نصب پروژه

```bash
cd /opt && git clone <REPO> vpn-shop && cd vpn-shop   # یا فایل‌ها را کپی کنید
npm install
cp .env.example .env
nano .env        # مقادیر پنل و رمز ادمین را وارد کنید
npm run db:seed  # ساخت دیتابیس + کاربر ادمین + پلن نمونه
npm run build
```

### ۳. اجرا با PM2

```bash
pm2 start deploy/ecosystem.config.js
pm2 save && pm2 startup   # اجرای خودکار بعد از ریبوت
```

### ۴. Nginx + SSL

```bash
sudo cp deploy/nginx.conf /etc/nginx/sites-available/vpn-shop
sudo nano /etc/nginx/sites-available/vpn-shop   # YOUR_DOMAIN را عوض کنید
sudo ln -s /etc/nginx/sites-available/vpn-shop /etc/nginx/sites-enabled/
sudo apt install -y certbot python3-certbot-nginx
sudo certbot --nginx -d YOUR_DOMAIN
sudo nginx -t && sudo systemctl reload nginx
```

### ۵. تنظیمات داخل سایت

با کاربر ادمین وارد شوید → **مدیریت → تنظیمات**:

| فیلد | توضیح |
|---|---|
| آدرس پنل 3x-ui | مثل `https://1.2.3.4:2053/AbCdEf` — **شامل مسیر پایه تصادفی پنل** |
| آدرس پایه Subscription | از تنظیمات پنل، بخش Subscription — مثل `https://1.2.3.4:2096/sub` |
| آدرس عمومی سرور | IP یا دامنه‌ای که کاربرها به آن وصل می‌شوند (برای لینک vless) |
| شماره کارت + به نام | اطلاعات کارت به کارت |

سپس دکمه **«آزمایش اتصال پنل»** را بزنید — باید لیست اینباندها را نشان دهد. شناسه اینباند موردنظر را در تعریف هر پلن (**مدیریت → پلن‌ها**) وارد کنید.

> **نکته پنل**: در تنظیمات 3x-ui حتماً Subscription را فعال کنید (پورت 2096 پیش‌فرض) و اگر گواهی TLS پنل خودامضاست، «پذیرش TLS خودامضا» را در تنظیمات سایت 1 بگذارید.

## جریان خرید

1. کاربر ثبت‌نام می‌کند و پلن را انتخاب می‌کند
2. صفحه پرداخت: شماره کارت + مبلغ یکتا (قیمت + کد چندتومانی) + مهلت ۴۵ دقیقه
3. کاربر رسید را آپلود می‌کند:
   - **رسید سالم** → کلاینت روی پنل ساخته می‌شود و لینک اشتراک فوراً تحویل داده می‌شود؛ رسید به صف بررسی مدیر می‌رود
   - **رسید مشکوک** (تکراری، سابقه جعل، ...) → بدون تحویل، منتظر تأیید مدیر می‌ماند
4. مدیر در **مدیریت → صف رسیدها**:
   - ✓ تأیید → وضعیت «فعال» نهایی می‌شود
   - ✕ رسید جعلی → سرویس روی پنل غیرفعال و سفارش لغو می‌شود (با دلیل، قابل بازگشت)
   - ✕ رسید نامشخص → کاربر می‌تواند رسید جدید بفرستد
